# AI聊天总结

一个用于微信聊天场景的 AI 聊天记录总结插件。
插件会读取当前聊天的近期消息，调用你配置的 AI 接口生成中文摘要，并将总结结果发送到文件传输助手。

## 主要功能

- 使用 `/ai 总结` 快速总结当前聊天。
- 使用 `/ai 总结 50`、`/ai 总结 120` 临时按条数总结。
- 支持自定义提示词模板。
- 模板支持按“最近天数”或“最近条数”提取消息。
- 支持设置默认模板。
- 支持根据 API 地址自动获取可用模型列表。
- 支持低消耗测试 API 可用性。
- 设置主面板已改为更清晰的卡片式界面。
- 总结结果默认发送到文件传输助手，避免误发到当前群聊或联系人。

## 使用命令

### 打开帮助

```text
/ai
```

或：

```text
/ai 帮助
```

### 打开设置

```text
/ai 配置
```

也可以使用：

```text
/ai 设置
```

设置面板中可以配置：

- API 地址
- API Key
- 模型名称
- 获取可用模型
- 测试 API 可用性
- 是否开启运行日志
- 提示词模板

> 设置主面板不再提供默认提取条数设置。  
> 如果需要指定条数，请使用 `/ai 总结 50` 这类命令，或在模板中设置按条数提取。

## 总结方式

### 使用默认模板总结

```text
/ai 总结
```

如果已经设置默认模板，会使用默认模板中的提示词和提取方式。

如果没有任何模板，则默认按最近 1 天聊天消息进行总结。

### 临时按条数总结

```text
/ai 总结 50
```

可用范围：

```text
50 ~ 200
```

例如：

```text
/ai 总结 120
```

表示临时总结最近 120 条可读消息。

### 使用指定模板总结

```text
/ai 总结 模板名
```

例如：

```text
/ai 总结 工作
```

插件会使用名为“工作”的模板进行总结。

## 模板说明

在 `/ai 配置` 中点击“新建模板”可以创建模板。

模板支持：

1. **模板名称**  
   用于 `/ai 总结 模板名` 调用。

2. **提示词内容**  
   用来控制 AI 总结风格、格式和重点。

3. **提取消息方式**  
   - 按最近天数提取
   - 按最近条数提取

4. **默认模板**  
   将某个模板设为默认后，发送 `/ai 总结` 会优先使用该模板。

## API 设置说明

本插件适配常见的 OpenAI-compatible messages 风格接口。

默认 API 地址为：

```text
https://api.openai.com/v1/chat/completions
```

你也可以填写兼容接口地址，例如第三方中转服务提供的 `/v1/chat/completions` 地址。

### 获取可用模型

在设置面板中填写：

- API 地址
- API Key

然后点击：

```text
获取可用模型并选择
```

插件会尝试根据 API 地址自动推导模型列表接口，例如 `/v1/models`。获取成功后，可以直接选择模型并自动填入“模型名称”。

### 测试 API 可用性

在设置面板中填写：

- API 地址
- API Key
- 模型名称

然后点击：

```text
测试API可用性
```

插件会发送一次低消耗测试请求：

- 内容：`ping`
- `max_tokens=1`

用于确认接口、Key 和模型是否可正常响应。

## 日志说明

设置面板中可以开启或关闭运行日志。

默认关闭日志。排查问题时可以临时开启。

为了降低崩溃和隐私风险，插件不会把完整聊天记录写入日志，只会记录必要的调试信息。

## 稳定性说明

插件已对按条数总结的历史消息读取方式做稳定性优化：

- 不再使用高频二分历史查询。
- 改为分段时间窗口逐步查询。
- 限制单次历史查询数量。
- 避免一次性读取过多历史消息导致微信崩溃。

如果在消息量特别大的群聊中使用按天数模板，建议不要设置过大的天数。

## 输出位置

总结完成后，结果会发送到：

```text
文件传输助手
```

不会直接发送到当前聊天，避免误发到群聊或联系人。
